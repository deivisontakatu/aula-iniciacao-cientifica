# 🧪 Atividade Prática — Organização do Projeto de IC (Kanban + GitHub)

Esta atividade tem como objetivo aplicar na prática os conceitos de **gestão de projetos científicos** utilizando **Kanban** e **versionamento com GitHub**, seguindo o modelo de Iniciação Científica apresentado em aula.

---

# 🎯 Objetivo da Atividade

- Estruturar um projeto de IC de forma organizada  
- Aplicar Kanban na gestão de tarefas acadêmicas  
- Organizar repositório técnico com boas práticas  
- Desenvolver rastreabilidade e documentação  

---

# 📌 Estrutura da Atividade

A atividade será dividida em **DUAS PARTES obrigatórias**:

---

## 🧩 PARTE 1 — Estruturação do Trello (Kanban)

Você deverá criar um quadro no Trello contendo as seguintes colunas:

- 📚 A Fazer  
- 🔄 Em Andamento  
- 👀 Em Revisão  
- ✅ Concluído  


---

### 🔹 Requisitos do Quadro

- Criar **no mínimo 20 cartões**
- Os cartões devem representar atividades reais da IC
- As tarefas devem estar distribuídas nas colunas
- O quadro deve estar organizado e legível

---

### 🔹 Sugestão de Cartões (Template)

#### 📚 A Fazer
- Definir tema da pesquisa  
- Delimitar problema científico  
- Formular hipótese  
- Definir objetivos gerais  
- Definir objetivos específicos  
- Criar repositório no GitHub  
- Configurar Mendeley ou Zotero  

---

#### 🔄 Em Andamento
- Realizar revisão bibliográfica  
- Buscar artigos científicos  
- Ler e fichar artigos  
- Definir metodologia  
- Modelar solução/protótipo  
- Definir variáveis do experimento  

---

#### 👀 Em Revisão
- Revisar problema científico  
- Validar hipótese  
- Revisar metodologia  
- Revisar referências  
- Revisar estrutura do projeto  

---

#### ✅ Concluído
- Tema definido  
- Problema validado  
- Hipótese definida  
- Metodologia estruturada  

---

## 💻 PARTE 2 — Estruturação do GitHub

Você deverá criar um repositório seguindo o padrão de organização apresentado no documento 03 - Método de Trabalho.

---

## 📸 Entrega

Você deverá enviar no grupo do WhatsApp:

- 📷 Print do quadro do Trello  
- 🔗 Link do repositório no GitHub (público)  

---

## ⚠️ Regras Importantes

- O trabalho deve conter as **duas partes obrigatórias**
- O Trello deve ter no mínimo **20 cartões**
- O GitHub deve estar **organizado e acessível**
- Trabalhos incompletos não serão considerados  

---

## 💡 Dica

Pense no seu projeto como um sistema real:

- O **Trello** mostra o andamento  
- O **GitHub** mostra a execução  

Se alguém olhar os dois, deve entender completamente seu projeto.
